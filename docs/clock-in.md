## 需求概述

本项目是一款基于微信生态开发的轻量化个人目标管理与习惯养成工具类小程序。针对当下用户在自我提升、长期规划落地过程中，普遍存在的「目标拆解无方法、执行过程易遗忘、进度追踪无感知、长期坚持难持续」的核心痛点，依托微信小程序免下载、易触达、强通知的生态优势，为用户提供极简、高效、闭环的目标全生命周期管理服务。


## 二、核心业务目标

搭建完整的「目标创建 - 拆解落地 - 提醒触达 - 打卡执行 - 进度追踪」全流程闭环，降低用户目标执行的门槛；
通过结构化的主子目标拆解体系、精准的定时通知机制、可视化的进度反馈，帮助用户提升目标完成率，养成稳定的执行习惯；
以单用户核心场景为基础，打造轻量化、无负担的产品体验，保障核心功能的流畅性与易用性，为后续迭代扩展奠定基础。


## 三、核心功能需求概述

一、需求补充说明
本次补充基于原需求框架，新增小程序关联微信公众号推送提醒全链路需求，核心解决微信小程序原生订阅消息的核心痛点：一次性订阅触达次数受限、长期订阅类目门槛高、用户关闭小程序后易漏看提醒。依托同主体微信认证服务号的稳定触达能力，构建双渠道、长效化的提醒体系，进一步保障打卡提醒的送达率，降低用户目标执行的遗忘率，全流程严格遵循微信平台运营规范，确保合规可用。
二、项目背景与产品定位
本项目是一款基于微信生态开发的轻量化个人目标管理与习惯养成工具类小程序。针对当下用户在自我提升、长期规划落地过程中，普遍存在的「目标拆解无方法、执行过程易遗忘、进度追踪无感知、长期坚持难持续」的核心痛点，依托微信小程序免下载、易触达的生态优势，联动同主体微信认证服务号的稳定消息触达能力，为用户提供极简、高效、闭环的目标全生命周期管理服务。

通过结构化的主子目标拆解体系、双渠道精准定时通知机制、可视化的进度反馈，帮助用户提升目标完成率，养成稳定的执行习惯；
打通小程序与同主体认证服务号的账号体系、消息触达链路，构建「小程序内消息 + 小程序订阅消息 + 公众号订阅通知」三位一体的高触达提醒体系，解决用户提醒漏看、触达失效的核心痛点；
以单用户核心场景为基础，打造轻量化、无负担的产品体验，保障核心功能的流畅性与易用性，为后续迭代扩展奠定基础。
四、核心功能需求概述
1. 主子目标体系管理
作为产品核心基础能力，支持用户完成目标的全生命周期管理：
支持主目标创建，可配置目标名称、详细描述、起止时间、分类标签（学习、工作、健康、生活等）、优先级等核心信息；
支持主目标的多级子目标拆解，可为子目标单独设置执行周期、截止时间、打卡规则，实现长期目标的可落地化拆分；
支持目标的全状态管理，包含进行中、已完成、已过期三种核心状态，同时提供目标的编辑、删除、状态一键变更等基础操作。
2. 打卡执行与进度追踪
实现目标执行的落地与正向反馈闭环：
支持子目标一键打卡操作，打卡时可补充文字备注、图片凭证等信息，完整留存执行记录；
自动基于子目标打卡完成情况，同步计算并更新主目标整体完成进度，通过进度条、完成率等形式直观可视化展示；
提供日历打卡视图，完整呈现用户月度打卡连续性与执行情况，打造打卡正向激励体系；
支持历史打卡记录的查询、筛选与回溯。
3. 公众号关联与推送提醒体系【本次新增核心模块】
3.1 小程序与公众号账号体系打通
基于微信开放平台能力，完成同主体小程序与微信认证服务号的关联绑定，通过 UnionID 实现用户账号唯一标识打通，确保小程序内设置的目标提醒规则，可精准匹配对应公众号用户，实现一对一精准推送；
实现用户账号无感绑定：用户完成公众号关注与推送授权后，自动完成小程序与公众号账号的绑定，无需用户重复注册、填写信息，全程低感知操作。
3.2 合规的公众号关注与推送授权引导
全流程严格遵循微信平台规则，不强制用户关注公众号即可使用小程序核心功能，仅将公众号推送作为提醒增强能力提供给用户；
多场景轻量化引导：在小程序首次启动、目标提醒设置、小程序订阅消息授权等核心场景，通过弹窗、固定入口等形式，正向引导用户关注公众号并开通推送权限，明确告知用户「公众号提醒更稳定，不会错过打卡通知，可随时关闭」；
明确的授权确认机制：开通公众号推送前，必须通过弹窗清晰告知用户推送的内容范围、推送场景、推送频次，获得用户主动、明确的同意后，方可开通推送权限，严禁静默授权、强制授权；
授权状态实时同步：小程序内设置专属入口，实时展示用户公众号关注状态、推送授权状态，支持用户一键开启 / 关闭全局公众号推送权限。
3.3 公众号推送场景与规则配置
推送场景与用户目标提醒全量对齐，覆盖核心业务场景，严禁推送与用户目标无关的内容：
固定周期打卡提醒：用户为目标设置的每日 / 每周固定时段的常规打卡提醒，同步推送公众号通知；
截止日期前置预警提醒：针对设置了截止时间的目标，按用户配置的规则（截止前 N 天 / N 小时）推送目标到期预警通知；
打卡遗漏补卡提醒：针对当日应打卡未完成的目标，在用户设置的兜底时段（如当日 22:00）推送遗漏打卡提醒，引导用户及时补卡；
目标完成正向激励通知：用户完成子目标 / 主目标时，推送完成祝贺通知，强化正向反馈，提升用户坚持意愿；
目标过期提醒：目标到达截止时间仍未完成时，推送目标过期通知，引导用户复盘或重启目标；
精细化推送配置能力：支持用户针对单个目标，独立设置是否开启公众号推送，可在全局开关的基础上，实现单目标的个性化管控；
推送频次合规管控：设置单用户单日推送上限，非必要场景不重复推送，避免过度打扰用户，同时严格符合微信公众平台推送频次规则。
3.4 推送内容与跳转链路优化
推送内容合规规范：使用微信公众平台审核通过的订阅通知模板，内容清晰包含目标名称、提醒事项、打卡时间，文案正向积极，与用户授权场景完全一致，严禁擅自变更模板用途、添加违规内容；
极简跳转链路：公众号推送通知点击后，可直接跳转到小程序对应目标的详情页 / 一键打卡页，用户点击后可直接完成打卡操作，最大程度缩短操作路径，提升打卡转化率；
兜底引导：针对用户长期未打开小程序的场景，推送内容可引导用户进入小程序，查看目标进度、完成打卡操作。
3.5 推送管理与数据监控
推送记录查询：在小程序提醒管理模块，支持用户查看历史公众号推送记录，包含推送时间、对应目标、推送状态（成功 / 失败）；
推送失败兜底机制：系统实时监控公众号推送结果，针对推送失败的场景，自动兜底触发小程序订阅消息 + 小程序内消息提醒，双渠道保障触达率；
后台数据统计能力：管理后台支持统计公众号关注转化率、推送触达率、消息点击跳转率、打卡转化率等核心数据，支撑推送策略的持续优化。
4. 全渠道定时通知提醒系统【原模块升级】
保障目标执行的触达率，解决用户执行遗忘的核心痛点，构建三位一体的全渠道提醒体系：
支持为单个目标自定义多维度提醒规则，包含固定周期提醒（每日 / 每周指定时段重复提醒）、截止日期前置提醒（截止前 N 天 / N 小时提醒）两大核心模式；
全渠道触达能力：
小程序内消息：小程序启动后，通过站内消息、红点提醒用户待打卡事项；
微信小程序订阅消息：严格遵循微信平台规范，引导用户授权后，通过小程序订阅消息推送提醒；
微信公众号订阅通知：基于新增的公众号推送体系，为授权用户推送稳定的长效提醒；
提供统一的提醒管理入口，支持用户对所有已设置的提醒进行一键启停、编辑、删除操作，可独立配置每个提醒的触达渠道。
5. 个人中心与数据概览
提供用户数据汇总与基础系统管理能力：
展示用户核心数据概览，包含进行中目标数、累计完成目标数、历史打卡统计等核心信息；
提供公众号推送管理专属入口，支持用户查看授权状态、管理推送权限、查看推送记录；
提供基础系统设置功能，包含用户授权管理、意见反馈、数据管理等配套能力。

五、非功能需求概述
易用性：产品操作路径极简，核心功能操作步骤不超过 3 步，界面设计简洁友好、积极正向，适配微信小程序全机型；公众号关注与授权流程清晰无歧义，用户操作成本低。
性能：页面加载响应迅速，核心打卡、目标创建操作无延迟，保证使用流畅度；公众号推送任务准时触发，推送延迟控制在合理范围内。
数据安全：优先采用微信云开发方案实现用户数据云端存储与跨设备同步，保障用户数据不丢失、不泄露；小程序与公众号账号数据互通严格遵循微信开放平台安全规范，不泄露用户隐私信息。
合规性【重点升级】：
严格遵循《微信小程序平台运营规范》《微信公众平台服务协议》《个人信息保护法》等相关法规与平台规则，所有消息推送、用户信息获取均需获得用户主动、明确授权后方可启用；
严禁强制用户关注公众号，不关注公众号不得限制小程序核心功能的使用；
公众号推送仅可使用平台审核通过的模板，内容必须与用户授权的目标提醒场景一致，严禁推送任何与用户目标无关的营销、广告内容；
为用户提供清晰、便捷的推送权限关闭入口，用户可随时一键关停所有公众号推送，保障用户的自主选择权；
严格管控推送频次，避免过度推送打扰用户，防止因用户投诉导致公众号、小程序被平台处罚。

六、目标用户与核心使用场景
核心目标用户：有自我提升、习惯养成、长期规划需求的学生群体、职场人士，以及有健身、阅读、备考等固定目标执行需求的个人用户。
核心使用场景：备考学习目标拆解与每日打卡、健身减脂计划周期执行与提醒、职场技能提升长期目标落地、早起 / 阅读等日常习惯养成打卡。
公众号推送核心场景：用户关闭小程序后，通过公众号推送稳定触达打卡提醒，避免用户遗忘当日打卡任务；目标即将到期时，通过公众号推送强提醒，督促用户完成目标收尾。

## 四、核心功能
一、 用户角色
由于是目标打卡类应用，初期通常聚焦于单用户场景，角色相对简单：
普通用户：使用小程序进行目标管理、打卡和接收通知的个人。
二、 核心功能需求（MVP）
1. 目标体系管理（主子目标）
这是本小程序的基础核心。
主目标创建：
输入：目标名称（如 “今年通过法考”）、描述、截止日期、优先级 / 标签（如工作、学习、健康）。
状态：进行中、已完成、已过期。
子目标拆解：
支持将主目标拆解为 N 个子目标（如 “看完《民法》”、“做完历年真题”）。
子目标可设置单独的截止日期或打卡频率（如每日、每周）。
CRUD 操作：支持对主 / 子目标进行编辑、删除、标记完成。
2. 打卡与进度追踪
快速打卡：针对子目标进行 “一键打卡”。
打卡记录：
支持添加文字备注（如 “今天学习了 2 小时”）。
支持上传图片凭证（可选）。
进度可视化：
主目标进度条（根据子目标完成情况自动计算）。
日历视图（显示当月打卡情况，形成 “打卡链”）。
3. 定时通知系统
这是用户明确要求的功能，关键在于提醒的灵活性。
提醒设置：
固定时间提醒：为特定目标设置每天 / 每周 X 点 X 分提醒。
智能提醒：例如 “截止日期前 3 天提醒”。
通知渠道：
小程序内消息通知。
微信订阅消息（为了保证触达率，这是必须的，需引导用户授权）。
提醒管理：用户可在列表中查看所有已设置的提醒，并可一键开启 / 关闭。
4. 数据概览（个人中心）
展示当前进行中的目标数量。
展示历史累计完成目标数。
基本的设置功能（如清空数据、意见反馈）。
三、 非功能需求
性能：页面加载速度快，打卡操作响应即时。
数据存储：
方案一：纯本地存储（微信小程序 Storage），开发快但换手机数据会丢。
方案二：云开发（微信云开发 / 自建后端），支持数据同步，推荐。
UI/UX：界面简洁、色调积极向上，操作路径不超过 3 步。
四、 扩展需求（未来迭代）
如果后续需要增强竞争力，可以考虑：
社交化：加入小组打卡、排行榜、互相监督。
模板库：提供 “考研备考”、“健身 30 天” 等预设模板，一键导入。
数据复盘：生成周报 / 月报，分析用户的打卡习惯和拖延症高发时段。
