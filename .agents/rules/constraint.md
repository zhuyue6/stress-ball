## 核心目标：让 AI 生成的代码严格遵循微信官方规范、贴合项目风格、保证性能与可维护性

### 一、角色定位
生成符合微信官方规范、项目风格统一、性能优异、可维护性强的代码，绝对不能生成不符合规范的内容。   

### 二、核心开发原则（必须严格遵守）  

官方规范优先：所有代码严格遵循《微信小程序官方开发文档》，不使用过时 API、不依赖非官方第三方框架（除非项目明确指定）。  
性能第一：优先考虑小程序的启动速度、运行流畅度，避免频繁 setData、避免过大的包体、避免冗余代码。  
治愈风设计：所有样式、交互必须贴合「马卡龙色系、柔和圆角、低饱和度、即时正反馈」的治愈风格，绝对不能出现高饱和度刺眼色彩、生硬交互。  
代码可维护性：代码结构清晰、命名规范、注释完整，方便后续迭代和团队协作。  
素材合规：所有美术、音效、字体必须标注来源，优先使用项目自有原创素材或有明确商用授权的素材，绝对不能生成无授权的素材引用代码。  

### 三、具体代码规范
#### 1. WXML 规范
语义化标签：优先使用 <view>、<text>、<image>、<button> 等官方语义化标签，避免过度嵌套 <view>。
数据绑定：使用 {{}} 进行数据绑定，绑定路径清晰，避免复杂表达式（复杂逻辑放在 JS 中处理）。
事件绑定：使用 bindtap、bindtouchstart 等官方事件绑定方式，避免使用 catchtap 除非明确需要阻止事件冒泡。
列表渲染：使用 wx:for 时必须指定 wx:key，优先使用列表项的唯一 ID 作为 key，避免使用 index（除非列表项顺序固定且不会变化）。
条件渲染：优先使用 wx:if（切换频率低），避免频繁切换时使用 hidden（切换频率高）。
#### 2. WXSS 规范
尺寸单位：优先使用 rpx（响应式像素），适配不同屏幕尺寸；固定尺寸使用 px，字体大小使用 rpx 或 px（根据设计稿）。
颜色规范：
主色调：马卡龙色系（如 #FFB6C1 浅粉、#87CEEB 浅蓝、#FFFACD 浅黄、#98FB98 浅绿）。
背景色：低饱和度浅色（如 #F5F5F5、#FAFAFA）。
文字色：深灰色（如 #333333、#666666），避免纯黑色（#000000）。
样式规范：
所有可点击元素必须有圆角（border-radius: 8rpx 及以上）。
所有容器元素优先使用柔和阴影（box-shadow: 0 4rpx 12rpx rgba(0,0,0,0.08)）。
避免使用高饱和度边框，优先使用浅色边框或无边框。
样式复用：使用 @import 引入公共样式文件（如 app.wxss、common.wxss），避免重复代码。
#### 3. JS 规范
模块化：公共逻辑（如音效管理、动画工具、数据存储）必须封装成独立模块，放在 utils/ 文件夹下，通过 module.exports 或 export 导出，在页面 / 组件中通过 require 或 import 引入。
数据管理：
页面数据放在 data 对象中，组件数据放在 data 或 properties 中。
避免频繁调用 setData，每次 setData 只更新必要的数据，避免一次性更新大量数据。
长列表数据使用「分页加载」或「虚拟列表」（如微信官方的 recycle-view 组件）。
API 使用：
优先使用微信小程序官方 API（如 wx.createInnerAudioContext、wx.createAnimation、wx.setStorageSync）。
使用 API 时必须加错误处理（如 fail 回调），避免小程序崩溃。
命名规范：
变量名、函数名使用小驼峰命名法（如 handleTouchStart、audioContext）。
常量名使用大写下划线命名法（如 MAX_COIN_COUNT、DEFAULT_SCENE）。
文件名使用小写下划线命名法（如 audio_manager.js、choose_page.wxml）。
注释规范：
所有函数必须加注释，说明函数功能、参数含义、返回值。
复杂逻辑必须加行内注释，说明逻辑思路。
公共模块必须加头部注释，说明模块功能、作者、更新时间。
#### 4. JSON 规范
页面配置：page.json 中只配置当前页面的专属配置（如 navigationBarTitleText、enablePullDownRefresh），全局配置放在 app.json 中。
组件配置：component.json 中必须设置 "component": true，如果需要使用其他组件，在 usingComponents 中引入。
格式规范：JSON 文件必须严格遵循 JSON 格式，不能有 trailing comma（ trailing comma 指最后一个属性后面的逗号），键名必须用双引号
