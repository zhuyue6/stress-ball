# 目标打卡（微信小程序）

轻量化个人目标管理与习惯养成工具，对应需求见仓库根目录 `docs/clock-in.md`。

## 前置条件（上线前请逐项确认）

| 项 | 说明 |
|----|------|
| 微信开发者工具 | 导入本目录 `apps/clock-in` |
| AppID | 在 `project.config.json` 中替换 `touristappid` 为你的小程序 AppID |
| 类目与审核 | 实际功能须与小程序注册类目一致 |
| 用户隐私 | 使用相册/相机能力需在后台配置《小程序用户隐私保护指引》 |
| 订阅消息 | 在 `utils/constants.js` 的 `SUBSCRIBE_TMPL_IDS` 填入公众平台审核通过的模板 ID；发送须在**服务端**调用且与用户场景一致 |
| 云开发（推荐） | 需求文档建议云端存储；当前 MVP 为本地 `wx.storage`，换机不同步，后续可替换 `utils/repository.js` |
| 公众号提醒 | 需开放平台同主体绑定、UnionID 与服务端模板推送；本仓库**仅**提供合规文案与「用户意愿」本地开关，不包含服务端 |

## 合规要点（实现约定）

- **不强制关注公众号**：核心流程（目标、子目标、打卡、日历）不依赖公众号。
- **订阅消息**：仅在用户主动操作（如勾选渠道并确认说明）时调用 `wx.requestSubscribeMessage`；拒绝不影响其他功能。
- **推送频次**：文案与常量中提示避免深夜推送；服务端实现时请遵守平台频次规则。
- **数据最小化**：默认不采集手机号等；公众号增强仅记录布尔偏好，便于你对接自有后端。

## 目录结构

- `pages/goals`：主目标列表、详情、编辑
- `pages/subgoal`：子目标编辑
- `pages/calendar`：月度打卡标记
- `pages/reminders`：提醒规则（站内 + 订阅消息意愿）
- `pages/mine`：概览、站内总开关、公众号偏好、清空数据
- `utils/repository.js`：本地数据层（可替换为云函数 / API）
- `utils/inAppReminder.js`：打开小程序时的站内提示（非推送）

## 开发命令

本包为原生小程序目录，无独立 `npm scripts`。用微信开发者工具打开即可预览。
